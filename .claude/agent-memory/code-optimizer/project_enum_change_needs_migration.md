name: enum-change-needs-migration
description: Editing a Drizzle pgEnum array does not generate a Postgres migration; ALTER TYPE ADD VALUE must be produced/run separately or inserts of the new value fail at runtime
metadata:
  type: project
---

Changing a `pgEnum(...)` value list in `database/schemas/*.schema.ts` (e.g. adding `reo` to `deal_type`) updates TypeScript/Zod types and passes `npm run check`, but does NOT by itself alter the live Postgres enum. The new value will be rejected by the DB on INSERT/UPDATE until an `ALTER TYPE ... ADD VALUE` migration is generated and pushed.

**Why:** `npm run check` only type-checks; it never touches the database. The drizzle migration journal (`database/drizzle/meta/_journal.json`) and snapshots are the source of truth for what the DB actually has, and they are not auto-regenerated by editing a schema file.

**How to apply:** When a review diff edits a `pgEnum` value array, verify a corresponding migration exists in `database/drizzle/` (check `git status` for a new `NNNN_*.sql` + updated snapshot, and that the latest snapshot's enum `values` include the new member). If absent, flag that `npm run db:push` / a generated migration is required before the change is safe in any non-empty DB. Also flag stale hand-maintained SQL like `database/all_database.sql` which is not auto-updated. Related: [[no-db-transactions]].
